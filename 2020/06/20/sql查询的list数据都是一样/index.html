sql查询的list数据都是一样

发表于 2020-06-20 | 分类于 problem
字数统计: 590 | 阅读时长 ≈ 2

| 工作不止，BUG不止

问题描述

​ 最近公司把我换到其他项目组（自助机）去了，听说采用的技术是最容易上手的springboot，本人感觉我应该能够胜任，毕竟咱的毕业设计可是用的是springboot，虽说工作之后就从来没有用过，但我对技术的要求就是差不多就可以上了（开玩笑的，毕竟咱编码人一直保持着严谨，求实的工作态度）。

​ 话回主题，采用技术springboot，spring data jpa ，像aop，adapter，advice，定时任务都是最基础的用法。数据源嘛有两个，一个是客户的副数据源（sql server），一个是本地的用于保存订单信息的主数据源(mysql)。当我使用不同的查询条件查询数据的时候，发现返回给我的list数据集合里面的实体对象竟然都是一样的，我用着同样的sql，同样的查询条件，查询出来的数据确实我想要的，这…我tm彻底的啥眼了。咱也是经过大风大浪的人，什么场面没有见过，第一步肯定就是要定位错误所在

嘛。

​ 当然快速定位到hibernate上，hibernate是不是有什么判断机制让他认为查询出的是同一条数据。于是，带着这个猜想，点开了我的实体类

嘿嘿，代码不在本地电脑上，代码下次补上

发现我的主键@Id放到了billno属性上，数据表也是使用的billno作为主键，但是数据表这个主键的数据是可以重复的，难道我发现了问题的所在。当我把实体类的主键换到一个记录自增的属性，感觉世界都对窝友好了许多。竟然查询出来了窝想要的结果

注意： 表中的主键与实体类中的主键不一定需要保持一致。实体类中的同@Id修饰主键在数据库中的数据应该是唯一的。如果查询出来的主键数据相同，那将都是第一条数据

解决方案

​ 第一条，更换实体类中的主键，选择一个不重复的字段充当主键

​ 第二条，使用联合主键@IdClass,添加复合主键
